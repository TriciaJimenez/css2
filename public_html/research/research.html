CSS Filter Properties

The filter property creates an extensible mechanism allowing special visual effects to be applied to content. There are currently three categories of filters: Visual filters and Reveal/Blend Transition filters. Multiple filters can be applied to a selector to produce interesting results, with the order of application often playing an important role in the final visual result. Current filters only apply in a visual context, but the extensibility of the property could allow for other capabilities.

Although this property is included in CSS 3, it is not yet part of the W3C standard. Most filters are supported only by Microsoft Internet Explorer 4.0+. However, Javascript can be used to create similar visual effects in other browsers like Firefox. If you include IE specific code for the filter property, you can validate your HTML code in W3C by by gathering them all in an IE specific style sheet and including the style sheet through a conditional statement. The validator and any browser other than Internet Explorer will parse it as regular comment, and thus just ignore it.

<!–[if IE]>
<link rel=”stylesheet” href=”filter.css” type=”text/css” media=”all” />
<![endif]–>

You can apply one or more filters to an element. The general format is:

filter: filtername1 (parameters) filtername2 (parameters) .. filternameN (parameters)

Here is an example:

<img src=”image.gif” mce_src=”image.gif” style=”filter: blur(strength=50) flipv()”>

alpha

The alpha filter is used to set the alpha transparency.

.filterTextAlpha { filter: alpha(opacity=40); }

The equivalent CSS3 property(what Firefox also uses) is opacity:x

blur

The blur filter is used to display a fuzzy effect.

.filterTextBlur { filter: blur(add = true, direction = 90, strength = 25); }

chroma

The chroma filter is used to specify the color for the transparent color.

.filterTextChroma { filter: chroma (color = purple); }
